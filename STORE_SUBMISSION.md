## Package URL

Partner Center expects a direct `https://` URL to a downloadable `.exe` or `.msi` file.

Use a URL like this:

```text
https://downloads.yourdomain.com/one-moment/0.1/One%20Moment%20Setup%200.1.0.exe
```

Requirements:

- Must be HTTPS
- Must point directly to the installer file
- Must not require a login
- Must not expire
- Must not be changed after submission
- Should be versioned in the path

Avoid using a GitHub release page URL or any URL that redirects through an HTML download page.

## Silent install note

`electron-builder` NSIS installers support silent installation. For Store submission, use:

```text
/S
```

If Microsoft asks for explicit per-user install context, keep the installer as user-scope and provide notes for certification.

## Current findings

- The generated installer is **not digitally signed**.
- Microsoft strongly recommends code signing for EXE/MSI submissions.
- Unsigned installers are much more likely to fail trust/security review or trigger warnings.

## Before submitting

1. Code-sign `One Moment Setup 0.1.0.exe` with a certificate trusted by Windows.
2. Upload the signed installer to a direct HTTPS file host.
3. Confirm the URL downloads the `.exe` immediately in a clean browser session.
4. Confirm the binary behind that URL never changes for that submission.
5. Enter `/S` in Installer parameters in Partner Center.
6. Add certification notes explaining that the app installs a Start menu entry and runs as a tray launcher.

## Signing command

If you have a `.pfx` certificate file:

```powershell
npm run sign:installer -- -PfxPath "C:\path\to\certificate.pfx" -Password "your-password"
```

If you have an installed certificate in Windows certificate store:

```powershell
npm run sign:installer -- -Thumbprint "YOUR_CERT_THUMBPRINT"
```
